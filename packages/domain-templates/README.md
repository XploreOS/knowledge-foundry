# @knowledge-foundry/domain-templates

Starter domain configurations for [Knowledge Foundry](https://github.com/XploreOS/knowledge-foundry): `generic`, `healthcare`, `legal`, `finance`, and `enterprise`. Each template is the seven YAML files every domain must define — `domain.yaml`, `taxonomy.yaml`, `source_policy.yaml`, `evidence_model.yaml`, `risk_rules.yaml`, `review_workflow.yaml`, `eval_questions.yaml`.

Consumed by `kf init-domain <id> --template <name>`; installed automatically as a dependency of `@knowledge-foundry/core`. Copy and adapt — templates are a starting point, not a finished domain.
